INSTALACIÓN DE SISTEMAS OPERATIVOS EN MÁQUINAS VIRTUALES

La instalación de muchos sistemas operativos se realiza de manera gráfica y requiere de la presencia física del servidor y el usuario. Esta instalación facilita en un gran porcentaje los pasos de instalación importantes como la asignación de tamaño y tipo a las particiones del sistema, configuración de red o instalación de paquetes básicos. En muchos casos los servidores no se encuentran en la misma red o en la misma locación geográfica que el usuario, lo cual haría imposible la instalación de un sistema operativo al servidor o una de sus máquinas virtuales. Teniendo en cuenta la problemática descrita anteriormente, administradores de sistemas orientados a servidores diseñaron la manera de conexión remota a estos servidores por medio de una herramienta web. Kanaka, un grupo de desarrollo github desarrollaron una consola web llamada console NoVNC; esta consola es una de las herramientas de más uso debido a que es muy sencilla y de fácil uso (Martin, 2014) NoVNC es un cliente VNC en HTML5 y Javascript, lo que permite su fácil renderización en la mayoría de navegadores modernos como Chrome, Mozilla Firefox y Opera (Velasco, 2015). También ofrece muchas ventajas como lo son:


  1. Funciona bajo cualquier navegador moderno y dispositivos móviles.
  2. Soporta cifrado WebSocket SSL/TLS.
  3. Permite usar el cursor local o el remoto (a preferencia del usuario).
  4. Emula muchas de las funcionalidades de una consola como Copiar, Cortar, Pegar, Scroll vertical y horizontal.

Los requerimientos mínimos para que la consola funcione en el navegador de su computadora son:

  1. Soporte HTML5 Canvas
  2. Soporte HTML5 WebSockets o Adobe Flash WebSockets.
  3. Motor Javascript para mejorar el rendimiento y velocidad a la consola NoVNC

Para este trabajo de grado se realizará la instalación de Centos 7 Minimal y Ubuntu Server 16.04 como dos de los sistemas operativos orientados a servidores más usados y modernos en la actualidad.
